Device Check

Run this on the phone you will carry before you walk in. Preflight tests the camera, sensors, crypto, and storage the device actually offers, then tells you which kit modules will run on it, so the magnetometer Flux needs or the secure context Exhibit needs is a known quantity, not an on-site surprise.

Capabilities

Presence checks run automatically. Camera, microphone, location, magnetometer, and motion need a tap to test, since they ask permission.

Readiness reflects the checks so far. Run the sensor tests on the Device tab for a true reading of camera, location, and the motion sensors.

How to read this

      

        Ready means the device has what the module needs. Limited means it will run with a feature reduced, like Lumen without an ambient light sensor or Touchstone without NFC. Will not run means a required capability is missing on this device.

        Most sensor and camera access needs a secure context. If the secure context check fails, host the kit over HTTPS or install it as an app before you rely on it.

        Magnetometer support is the usual gap. Where the dedicated sensor is absent, Preflight falls back to the orientation sensor and marks Flux as limited.
